## 2024-05-18 - Prevent SQL Injection via Direct Interpolation
**Vulnerability:** Found a SQL injection vulnerability in `sago-core/src/postgres.rs` where an `identifier` passed to `PostgresSchemaProvider::get_data()` was directly string formatted into a SQL query: `format!("SELECT * FROM {}", identifier)`.
**Learning:** This repo builds database integrations dynamically, receiving external identifiers (e.g. from `.toml` config like `tables = ["users"]`). Since it supports multi-part identifiers (like `schema.table`), standard approaches of just splitting and defaulting to `"public"` can override native database `search_path` resolution.
**Prevention:** Rather than trying to parse or default schemas explicitly, correctly iterate through all parts of a multi-part identifier (split by `.`), escape double quotes (`.replace('"', "\"\"")`), wrap each part in double quotes, and join them back together. This supports native database capabilities while strictly neutralizing SQL injection vectors.
